OC(2) 对 C 的扩展

2016/9/23 19:11 下午 posted in  ObjC comments  

解构 Hello, Objective-C

#import  <Foundation/Foundation.h>
int main(int argc, const char * argv[]) {
   NSLog(@"Hello, Objective-C.")
   return 0;
}

#import

  • 查询 Foundation 框架中的 Foundation.h 头文件,使用# import可以避免重复包含
  • Foundation.h 涵盖100多个文件.但 Xcode 使用预编译头文件加快读取速度
  • Cocoa 的组成部分有 Foundation 和 AppKit ,导入 Foundation, 就可以使用 Cocoa 特性写代码了
  • Foundation 框架: 处理 UI 之下的那些层的特性,比如数据结构和通信机制

NSLog() 和@ "字符串"

  • 接受格式说明符 (比如 %d)
  • NS前缀: NextSTEP工具包
  • @符号意味着符号内的字符串作为 Cocoa 的 NSString 元素来处理

布尔类型

C 语言拥有 bool 数据类型,该类型有 true 和 false 两个值,Objective-C 提供了一个相似的 BOOL, 它具有 YES 和 No 两个值

BOOL

  1. 带符号的字符类型 (signed char)
  2. 使用8位的存储空间
  3. 通过# define 把 YES 定义为 1, NO 定义为 0
  4. 不小心把大于1的整型赋值给 BOOL 变量,最低字位会被用作 BOOL 值

通过实例来理解 BOOL 类型

#import <Foundation/Foundation.h>

BOOL areIntsDifferent(int a, int b){
    if (a == b) {
        return (NO);
    }else{
        return (YES);
    }
    
}
NSString * boolString(BOOL str){
    if (str == YES) {
        return (@"YES");
    }else{
    return (@"NO");
    }
    
}
int main(int argc, const char * argv[]) {
    BOOL areInts=areIntsDifferent(5, 5);
    NSLog(@"are %d and %d different? %@\n",5,5,boolString(areInts));    // 输出 NO
    
    areInts=areIntsDifferent(10, 0);
    NSLog(@"are %d and %d different? %@\n",10,0,boolString(areInts));    //输出 YES
    return 0;
}

获取布尔值

下面这种方式获取布尔值需要注意,由于 BOOL 值有8位的存储空间的,因此通过判断 BOOL 值为 YES 的话,会出现其他值

然而,直接与 NO 比较是安全的, 因为 C 语言中的假值只有一个 0

BOOL areIntsDifferent_faulty(int a, int b){
    return (a-b);
}
int main(int argc, const char * argv[]) {
    BOOL b_value=areIntsDifferent_faulty(5, 10);
    NSLog(@"BOOL %d and %d is %d\n",5,10,b_value);   //BOOL 5 and 10 is -5
    
    b_value=areIntsDifferent_faulty(10, 10);
    NSLog(@"BOOL %d and %d is %d\n",10,10,b_value);  //BOOL 10 and 10 is 0
}

NSString

boolString() 的返回值类型是一个指向 NSString 的指针,意味着会返回一个 Cocoa 字符串.(原因摘自stackoverflow,关于指针使用的原因,还有图看堆栈)

You use a pointer always when referring to something on the heap and sometimes, but usually not when referring to something on the stack.

Since Objective-C objects are always allocated on the heap,you always use pointers to Objective-C objects.
